## Cycle 1 brief

**Objective:** Stand up a deployable Next.js app on Vercel with the Wenger-branded login screen rendering correctly.

**Scope (what's IN):**
- Initialize Next.js 15 with App Router, TypeScript strict mode, Tailwind CSS v4
- Configure brand colors as CSS custom properties in `globals.css` (Wenger navy `#0A3758`, dark variant `#082a45`, light variant `#1a5685`, warm accent `#b8612a`)
- Place both logos in `/public/` (`logo-white.png` for dark backgrounds, `logo-blue.png` for light backgrounds)
- Build the login page (`app/page.tsx`):
  - Full-bleed gradient background, linear from `#082a45` (top) to `#1a5685` (bottom)
  - White Wenger logo centered horizontally, ~80px tall, positioned in the upper third
  - "Field Notes" wordmark below the logo in Georgia serif, white, italic, ~18px
  - Subtle subtitle: "Pick your name to begin"
  - 6 rep buttons (Brooke, Jackie, Rahki, Chad, Tam, Linda) in a responsive grid (2 cols mobile, 3 cols on wider screens)
  - Rep buttons: glassmorphism style (`bg-white/10 backdrop-blur-md border border-white/20`), hover lifts subtly, smooth 200ms transition
  - Selecting a rep navigates to `/placeholder` (a temporary page that just confirms the rep name and shows "Cycle 2 will build the app shell here")
- Smooth fade-in animation on the login card mounting
- Mobile-first: must look excellent on 375px width, scale up gracefully

**Scope (what's OUT — defer to future cycles):**
- No real authentication (Cycle 6)
- No database (Cycle 5)
- No map, no form, no admin (Cycles 3, 4, 7)
- No app shell — just the placeholder destination page (Cycle 2)
- No AI integration (Cycle 8)

**Done when:**
- Local `npm run dev` runs without errors
- `npm run build` succeeds with no TypeScript errors
- Login page renders correctly at 375px (mobile) and 1280px (desktop)
- Selecting any rep navigates to `/placeholder` and that page shows the selected rep's name
- Code pushed to `main` on GitHub
- Vercel auto-deploy succeeds
- Live URL shows the login screen looking exactly as intended

## Execution steps (after user approval)

1. **Scaffold the Next.js project** using `npx create-next-app@latest .` with these flags:
   - TypeScript: yes
   - ESLint: yes
   - Tailwind CSS: yes
   - `src/` directory: yes
   - App Router: yes
   - Turbopack: yes
   - Customize import alias: no (use default `@/*`)
2. **Verify scaffold works** — `npm run dev`, confirm default page loads. Commit checkpoint: `chore: initial Next.js scaffold`.
3. **Add brand tokens** to `src/app/globals.css`:
   ```css
   :root {
     --brand-navy: #0A3758;
     --brand-navy-dark: #082a45;
     --brand-navy-light: #1a5685;
     --brand-warm: #b8612a;
     --brand-warm-soft: #f5ebe0;
   }
   ```
   Also configure Tailwind to expose these as utility classes (`bg-brand-navy`, `text-brand-warm`, etc.) — modify `tailwind.config.ts` or use Tailwind v4's `@theme` directive in CSS, whichever is the current convention for the installed Tailwind version.
4. **Build the login page** at `src/app/page.tsx`:
   - Use the `next/image` component for the logo
   - Implement the gradient background with Tailwind: `bg-gradient-to-b from-[#082a45] to-[#1a5685]` (or via CSS variables)
   - Build the rep grid with the 6 names hardcoded for now
   - On rep selection, use `next/navigation` `useRouter` to push to `/placeholder?rep=NAME`
   - Add a fade-in animation using Tailwind's `animate-in` utilities or a simple CSS transition
5. **Build the placeholder page** at `src/app/placeholder/page.tsx`:
   - Reads `rep` from `useSearchParams`
   - Shows: "Hi, [rep] — Cycle 2 will build the app shell here."
   - Includes a "Back to login" link
6. **Mobile-test locally**: open Chrome DevTools, set viewport to 375px, walk through login → placeholder. Verify smooth.
7. **Build-test**: `npm run build` succeeds with no errors.
8. **Commit checkpoint**: `feat(cycle-1): branded login page with rep selector`.
